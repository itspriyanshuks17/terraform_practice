CLI Reference

The Terraform workflow, short enough to keep open while practicing.

Use this page while working through the hands-on AWS labs. It keeps the important commands close without digging through the full notes.

Workflow

Standard command loop

01

terraform init

Initialize providers and backend configuration before any other command.

->
02

terraform fmt

Format Terraform files to the standard HCL style.

->
03

terraform validate

Check syntax and configuration validity before planning.

->
04

terraform plan

Preview what Terraform will create, update, or destroy.

->
05

terraform apply

Execute the reviewed plan and create real infrastructure.

->
06

terraform destroy

Delete the managed lab resources when practice is complete.

->
Daily Terraform commands
terraform init
terraform fmt -recursive
terraform validate
terraform plan -out=tfplan
terraform apply tfplan
terraform output
terraform show
terraform destroy
Situation Command Use carefully because
Inspect state resources terraform state list Useful before imports, moves, and cleanup.
Import an existing resource terraform import ADDRESS ID The config still has to match the imported object.
Move state address terraform state mv OLD NEW Great for refactors, but check addresses first.
Remove from state only terraform state rm ADDRESS It does not delete the real cloud resource.
Release stuck lock terraform force-unlock LOCK_ID Only use when you know no other apply is running.
Practice order: finish EC2/S3 first, then VPC/Nginx, then static website plus remote backend. That path matches the way the concepts build on each other.
